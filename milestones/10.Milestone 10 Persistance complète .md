## Concepts Python mis en pratique

| Concept | API sqlite3 | Usage dans Al Qalam |
|---|---|---|
| Connexion | `sqlite3.connect(chemin)` | Ouverture/création du fichier `.db` |
| Row factory | `conn.row_factory = sqlite3.Row` | Accès résultat par nom de colonne |
| PRAGMA | `PRAGMA foreign_keys`, `journal_mode = WAL` | Intégrité + performances |
| Context manager | `with conn as conn:` | Commit auto, rollback sur exception |
| DDL idempotent | `CREATE TABLE IF NOT EXISTS` | Schéma créé une seule fois |
| Index | `CREATE INDEX IF NOT EXISTS` | Filtres rapides sur date/ref/type |
| Paramètres nommés | `VALUES (:ref, :nom, ...)` | Protection injection SQL |
| Paramètres positionnels | `WHERE ref = ?` | Requêtes dynamiques |
| Agrégation | `COUNT(*), SUM(qte) GROUP BY type_mvt` | Stats sans boucle Python |
| Multi-instructions | `conn.executescript("""...""")` | Création schéma complet en une fois |
| Migration | lecture JSON → INSERT → renommage `.bak` | Transition sans perte de données |

## Schéma de base de données

```sql
-- Table catalogue — un enregistrement par produit
CREATE TABLE IF NOT EXISTS produits (
    ref        TEXT PRIMARY KEY,
    nom        TEXT NOT NULL,
    categorie  TEXT NOT NULL,
    prix_achat REAL NOT NULL CHECK(prix_achat >= 0),
    prix_vente REAL NOT NULL CHECK(prix_vente >= 0),
    qte        INTEGER NOT NULL DEFAULT 0 CHECK(qte >= 0),
    seuil_min  INTEGER NOT NULL DEFAULT 5 CHECK(seuil_min >= 0)
);

-- Table historique — un enregistrement par mouvement
CREATE TABLE IF NOT EXISTS mouvements (
    id        INTEGER PRIMARY KEY AUTOINCREMENT,
    date      TEXT NOT NULL,          -- ISO 8601 : "2026-03-10T14:32:05"
    type_mvt  TEXT NOT NULL,          -- "entree" | "sortie" | "ajustement" | "retour"
    ref       TEXT NOT NULL,          -- référence produit concerné
    produit   TEXT NOT NULL,          -- nom produit (dénormalisé pour lisibilité)
    qte       INTEGER NOT NULL CHECK(qte > 0),
    note      TEXT DEFAULT ''
);

-- Index de performance pour les filtres courants
CREATE INDEX IF NOT EXISTS idx_mvt_date    ON mouvements(date);
CREATE INDEX IF NOT EXISTS idx_mvt_ref     ON mouvements(ref);
CREATE INDEX IF NOT EXISTS idx_mvt_type    ON mouvements(type_mvt);
```

### 2. `services/database_service.py` — DatabaseService (nouveau)

Couche d'accès aux données 100% SQLite. Toutes les opérations passent par cette classe.

#### Connexion configurée

```python
def _connexion(self) -> sqlite3.Connection:
    conn = sqlite3.connect(str(self._chemin))
    conn.row_factory = sqlite3.Row          # accès résultat["ref"] au lieu de résultat[0]
    conn.execute("PRAGMA foreign_keys = ON")
    conn.execute("PRAGMA journal_mode = WAL")   # Write-Ahead Logging
    return conn
```

#### Requêtes filtrées dynamiques

```python
def charger_mouvements(self, type_mvt=None, ref=None,
                        date_debut=None, date_fin=None, limite=None):
    conditions, params = [], []
    if type_mvt:   conditions.append("type_mvt = ?"); params.append(type_mvt)
    if ref:        conditions.append("ref = ?");       params.append(ref)
    if date_debut: conditions.append("date >= ?");     params.append(date_debut)
    if date_fin:   conditions.append("date <= ?");     params.append(date_fin + "T23:59:59")

    clause_where = ("WHERE " + " AND ".join(conditions)) if conditions else ""
    clause_limit = f"LIMIT {limite}" if limite else ""

    sql = f"SELECT * FROM mouvements {clause_where} ORDER BY date DESC {clause_limit}"
    with self._connexion() as conn:
        return [dict(row) for conn.execute(sql, params).fetchall()]
```

#### Agrégation par type

```python
def stats_mouvements(self) -> dict:
    with self._connexion() as conn:
        rows = conn.execute(
            "SELECT type_mvt, COUNT(*) as nb, SUM(qte) as qte_totale "
            "FROM mouvements GROUP BY type_mvt"
        ).fetchall()
    # → {"entree": {"nb": 12, "qte_totale": 450}, "sortie": {...}, ...}
```

#### Migration JSON → SQLite

```python
def migrer_depuis_json(self, chemin_json: Path) -> int:
    """Appelée au 1er lancement : importe stock.json et le renomme en .bak."""
    if not chemin_json.exists(): return 0
    # Ne migrer que si la table est vide
    if conn.execute("SELECT COUNT(*) FROM produits").fetchone()[0] > 0: return 0

    data = json.loads(chemin_json.read_text(encoding="utf-8"))
    for d in data:
        self.inserer_produit(d)
    chemin_json.rename(chemin_json.with_suffix(".json.bak"))
    return len(data)
```

## Points clés SQLite à retenir

### Toujours utiliser des paramètres, jamais de f-string SQL

```python
# ✅ Correct — paramètre ? protège contre les injections SQL
conn.execute("SELECT * FROM produits WHERE ref = ?", (ref,))

# ❌ Dangereux — injection SQL possible si ref contient du SQL
conn.execute(f"SELECT * FROM produits WHERE ref = '{ref}'")
```

### Le context manager `with conn` gère les transactions

```python
# Commit automatique si aucune exception
# Rollback automatique en cas d'exception
with sqlite3.connect("alqalam.db") as conn:
    conn.execute("UPDATE produits SET qte=? WHERE ref=?", (50, "CRAY-001"))
    conn.execute("INSERT INTO mouvements ...")
    # → si une des deux échoue, AUCUNE des deux n'est persistée (atomicité)
```

### `row_factory = sqlite3.Row` pour un accès lisible

```python
conn.row_factory = sqlite3.Row

row = conn.execute("SELECT ref, nom FROM produits WHERE ref=?", ("CRAY-001",)).fetchone()

# Avec row_factory :
print(row["ref"])    # "CRAY-001"  ← accès par nom
print(row["nom"])    # "Crayon HB"

# Sans row_factory (par défaut) :
print(row[0])        # "CRAY-001"  ← accès par index seulement
```

### WAL mode pour de meilleures performances

```python
conn.execute("PRAGMA journal_mode = WAL")
# Write-Ahead Logging : les lectures et écritures ne se bloquent plus mutuellement
# Recommandé pour toute application avec lectures fréquentes
```
