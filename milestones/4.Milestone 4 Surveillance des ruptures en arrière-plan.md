## Concepts threading mis en œuvre

| Concept                | Classe / méthode                           | Rôle dans le projet                            |
| ---------------------- | ------------------------------------------ | ---------------------------------------------- |
| **Thread daemon**      | `threading.Thread(daemon=True)`            | S'arrête automatiquement avec l'app            |
| **Arrêt propre**       | `threading.Event` · `.set()` · `.wait(t)`  | Signal d'arrêt sans `time.sleep`               |
| **Exclusion mutuelle** | `threading.Lock` · `with self._lock:`      | Protège `_produits` contre les race conditions |
| **Communication**      | `queue.Queue` · `.put()` · `.get_nowait()` | Thread → UI sans bloquer                       |
| **Polling UI-safe**    | `widget.after(500, callback)`              | Seul le thread principal modifie les widgets   |

## Les règles d'or du threading Tkinter

```
Thread de surveillance              Thread principal (Tkinter)
        │                                     │
        │  queue.Queue.put(alerte)             │
        │ ─────────────────────────────────►  │
        │                                     │  widget.after(500, poll)
        │                                     │  ┌─ queue.get_nowait() ─┐
        │                                     │  │  tree.insert(...)    │  ← OK ✅
        │                                     │  └──────────────────────┘
        │
        │  label.configure(text="...")        ← INTERDIT ❌
```

**Règle 1** : Ne jamais modifier un widget depuis un thread secondaire.
**Règle 2** : Utiliser `queue.Queue` pour passer les données au thread principal.
**Règle 3** : Utiliser `after()` côté UI pour lire la queue sans bloquer.

## Guide de réalisation étape par étape

### Étape 1 — Protéger StockService avec un Lock (15 min)

Dans `services/stock_service.py`, ajoutez un `threading.Lock` dans `__init__` et protégez toutes les méthodes qui lisent ou écrivent `_produits`.

**Piège à éviter** : Si `methode_A()` acquiert le Lock et appelle `methode_B()` qui acquiert aussi le Lock → **deadlock** !

```python
# ❌ DEADLOCK : Lock non réentrant
def entree_stock(self, ref, qte, note=""):
    with self._lock:
        produit = self.get_produit(ref)   # get_produit() essaie d'acquérir _lock → bloqué !
        produit.qte += qte

# ✅ CORRECT : accès direct sans appel imbriqué
def entree_stock(self, ref, qte, note=""):
    with self._lock:
        if ref not in self._produits:
            raise KeyError(f"Produit '{ref}' introuvable.")
        produit = self._produits[ref]     # accès direct, pas d'appel à get_produit()
        produit.qte += qte
```

### Étape 2 — Créer SurveillanceService (20 min)

```python
# services/surveillance_service.py
import threading, queue
from datetime import datetime

class SurveillanceService:
    def __init__(self, stock_service, intervalle=30):
        self._stock      = stock_service
        self._intervalle = max(5, int(intervalle))
        self._stop_event = threading.Event()   # signal d'arrêt
        self._lock       = threading.Lock()    # protège _historique
        self._file       = queue.Queue()       # canal vers l'UI
        self._historique = []
        self._thread     = None

    def demarrer(self):
        self._stop_event.clear()
        self._thread = threading.Thread(
            target=self._boucle,
            name="SurveillanceStock",
            daemon=True,           # s'arrête avec l'application
        )
        self._thread.start()

    def arreter(self):
        self._stop_event.set()     # réveille le .wait() dans la boucle
        if self._thread and self._thread.is_alive():
            self._thread.join(timeout=5)

    def _boucle(self):
        while not self._stop_event.is_set():
            self._verifier()
            self._stop_event.wait(timeout=self._intervalle)  # libère le CPU

    def _verifier(self):
        with self._lock:
            produits = self._stock.produits_en_alerte()
        if produits:
            alerte = {"timestamp": datetime.now().isoformat(), "produits": produits}
            self._file.put(alerte)              # thread-safe
            with self._lock:
                self._historique.append(alerte)

    def lire_alertes(self):
        """Vide la queue — appelé depuis l'UI via after()."""
        resultats = []
        try:
            while True:
                resultats.append(self._file.get_nowait())
        except queue.Empty:
            pass
        return resultats

    @property
    def est_actif(self):
        return self._thread is not None and self._thread.is_alive()
```

### Étape 3 — Créer AlertesFrame (20 min)

```python
# ui/frames/alertes_frame.py
import customtkinter as ctk
from tkinter import ttk

class AlertesFrame(ctk.CTkFrame):
    def __init__(self, parent, surveillance_service, **kwargs):
        super().__init__(parent, **kwargs)
        self._svc = surveillance_service
        self._id_poll = None
        self._construire_ui()
        self._demarrer_polling()

    def _demarrer_polling(self):
        # after() = planification sur le thread principal → UI-safe
        self._id_poll = self.after(500, self._poll)

    def _poll(self):
        alertes = self._svc.lire_alertes()      # lecture non-bloquante
        for a in alertes:
            self._inserer(a)                    # modification widget OK ici
        self._id_poll = self.after(500, self._poll)   # replanifier

    def _inserer(self, alerte):
        # ... ajouter une ligne dans le Treeview ...
        pass

    def arreter_polling(self):
        """Appelé à la fermeture — évite un after() orphelin."""
        if self._id_poll:
            self.after_cancel(self._id_poll)
```
