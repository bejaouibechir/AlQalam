## Concepts pytest mis en pratique

| Concept | Usage dans Al Qalam |
|---|---|
| `@pytest.fixture` | Setup réutilisable : `produit_demo`, `db_vide`, `stock_vide` |
| `scope="function"` (défaut) | Fixture recréée pour chaque test → isolation totale |
| `tmp_path` | Dossier temporaire fourni par pytest → fichiers CSV/DB propres |
| `monkeypatch` | Redirige `DB_PATH` vers une base temporaire pendant le test |
| `yield` dans une fixture | Setup avant le yield, teardown automatique après |
| `pytest.raises` | Vérifie qu'une exception est bien levée |
| `pytest.raises(match=)` | Vérifie aussi le message d'erreur |
| `@pytest.mark.parametrize` | Même test avec N jeux de données (39 cas dans `test_validators.py`) |
| `pytest.approx` | Comparaison de flottants sans erreur d'arrondi |
| Classes `Test*` | Regroupement logique des tests (pas de `setUp`/`tearDown`) |
| `conftest.py` | Fixtures partagées entre tous les fichiers de test |
| `pytest.ini` | Configuration : `testpaths`, `addopts`, `[coverage:run]` |
| `pytest-cov` | Mesure de couverture avec `--cov` + `--cov-report=term-missing` |

## Détail des fichiers de test

### `conftest.py` — Fixtures partagées

```python
@pytest.fixture
def produit_demo() -> Produit:
    """Produit valide réutilisé dans plusieurs fichiers de test."""
    return Produit("CRAY-001", "Crayon HB", "Ecriture", 0.15, 0.50, 100, 20)

@pytest.fixture
def db_vide(tmp_path):
    """DatabaseService avec base SQLite temporaire — isolée par test."""
    from services.database_service import DatabaseService
    return DatabaseService(tmp_path / "test.db")

@pytest.fixture
def stock_vide(tmp_path, monkeypatch):
    """
    StockService avec Singleton réinitialisé + DB temporaire.
    Problème : StockService est un Singleton → risque de contamination entre tests.
    Solution :
      1. SingletonMeta._reset() — efface le registre avant le test
      2. monkeypatch.setattr — redirige DB_PATH vers tmp_path/test_stock.db
      3. importlib.reload — recharge stock_service pour qu'il lise le bon DB_PATH
      4. yield — fournit le service au test
      5. teardown (après yield) — SingletonMeta._reset() + reload
    """
    SingletonMeta._reset()
    monkeypatch.setattr(config, "DB_PATH", tmp_path / "test_stock.db")
    importlib.reload(sm)
    svc = sm.StockService()
    yield svc
    SingletonMeta._reset()
    importlib.reload(sm)
```

    (5,  10, True),   # stock ≤ seuil

## Bug détecté et corrigé par les tests

Le test `test_export_mouvements_apres_operation` a révélé un bug dans `csv_service.py` (hérité de V8) :

```python
# ❌ Bug V8-V10 — AttributeError : 'EntreeMouvement' n'a pas d'attribut 'ref'
nom_produit = self._stock.get_produit(m.ref).nom

# ✅ Correction V11 — l'attribut correct est ref_produit
ref_prod    = m.ref_produit
nom_produit = self._stock.get_produit(ref_prod).nom
```

C'est exactement le rôle des tests : **détecter les régressions et les incohérences d'interface**.

## Commandes pytest essentielles à retenir

| Commande | Effet |
|---|---|
| `pytest -v` | Mode verbeux — affiche le nom de chaque test |
| `pytest -q` | Mode silencieux — résumé compact |
| `pytest --tb=short` | Traceback court en cas d'échec |
| `pytest -k "entree"` | Lance uniquement les tests dont le nom contient "entree" |
| `pytest -x` | Arrête au 1er échec |
| `pytest --lf` | Relance uniquement les tests qui ont échoué lors du dernier run |
| `pytest --co` | Liste les tests collectés sans les exécuter |
| `pytest --cov=services` | Mesure la couverture du dossier `services/` |
| `pytest --cov-report=html` | Rapport HTML interactif dans `htmlcov/` |
