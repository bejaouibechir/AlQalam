## Objectif du sprint

Connecter **Al Qalam Stock Manager** au monde extérieur via des fichiers CSV.
À la fin de ce sprint, l'application peut :

- **Importer** un catalogue fournisseur CSV → créer ou mettre à jour les produits
- **Exporter** le catalogue complet → partager avec un partenaire ou sauvegarder
- **Exporter les mouvements** → audit comptable, contrôle de gestion
- **Exporter un rapport comptable** → valeur stock, marges unitaires et totales

## 1. Pourquoi `csv` et pas `pandas` ici ?

```
Module csv (standard)               pandas (externe)
──────────────────────────────────  ─────────────────────────────────
✅ Intégré Python — aucun pip       ❌ pip install pandas requis
✅ Mémoire constante (streaming)    ❌ charge tout en RAM
✅ Parfait pour import/export       ✅ Meilleur pour analyse/stats
✅ Contrôle total ligne par ligne   ✅ Filtres/agrégations en 1 ligne
✅ Pas de dépendance externe        ✅ Idéal pour > 10 000 lignes
```

**Règle de décision pour Al Qalam :**
- Import fournisseur < 1 000 lignes → `csv` (léger, aucune dépendance)
- Analyse de tendances > 10 000 mouvements → `pandas` (sprint futur)

### 3.1 Structure de la classe

```python
class ErreurCsv(Exception):
    """Levée quand un fichier CSV ne respecte pas la structure attendue."""


class CsvService:
    """
    Service d'import/export CSV pour Al Qalam.

    [V8] Utilise uniquement le module standard `csv` :
      - Aucune dépendance externe (pas de pandas)
      - Traitement ligne par ligne → consommation mémoire constante
      - Compatible Excel grâce à encoding="utf-8-sig" (BOM)

    Usage :
        service = CsvService(stock_service)
        rapport = service.importer_catalogue("fournisseur.csv")
        service.exporter_comptabilite("rapport_mars.csv")
    """

    def __init__(self, stock_service):
        self._stock = stock_service   # injection de dépendance (pas un Singleton)
        CSV_DIR.mkdir(parents=True, exist_ok=True)
```

`CsvService` **n'est pas un Singleton** — il peut y avoir plusieurs instances
(une par fenêtre de dialogue par exemple). Il **dépend** de `StockService` (Singleton V6).

### 3.2 Import — `importer_catalogue()`

```python
@journaliser("import_csv")
def importer_catalogue(self, chemin: str | Path) -> dict:
    """
    Importe un catalogue fournisseur CSV dans le stock.

    Comportement :
      - ref absente du stock  → ajouter_produit() avec qte=0
      - ref déjà présente    → mettre_a_jour_produit() (prix + seuil)
      - ligne invalide        → ajoutée à erreurs[], traitement continue

    Returns:
        dict {
            "importes"  : nb de nouveaux produits créés,
            "mis_a_jour": nb de produits mis à jour,
            "erreurs"   : liste de str décrivant les lignes rejetées,
            "total_lues": nb total de lignes traitées
        }
    """
    chemin = Path(chemin)
    if not chemin.exists():
        raise FileNotFoundError(f"Fichier introuvable : {chemin}")

    rapport = {"importes": 0, "mis_a_jour": 0, "erreurs": [], "total_lues": 0}

    with open(chemin, "r", encoding="utf-8-sig") as f:
        lecteur = csv.DictReader(f)

        # ── 1. Valider la structure AVANT de traiter ─────────────────────
        colonnes_presentes = set(lecteur.fieldnames or [])
        manquantes = set(COLONNES_CATALOGUE_IMPORT) - colonnes_presentes
        if manquantes:
            raise ErreurCsv(
                f"Colonnes manquantes dans {chemin.name} : "
                f"{', '.join(sorted(manquantes))}"
            )

        # ── 2. Traitement ligne par ligne ────────────────────────────────
        for numero_ligne, ligne in enumerate(lecteur, start=2):
            rapport["total_lues"] += 1
            try:
                ref        = ligne["ref"].strip()
                nom        = ligne["nom"].strip()
                categorie  = ligne["categorie"].strip()
                prix_achat = float(ligne["prix_achat"].replace(",", "."))
                prix_vente = float(ligne["prix_vente"].replace(",", "."))
                seuil_min  = int(ligne["seuil_min"])

                if not ref or not nom or not categorie:
                    raise ValueError("ref, nom et categorie ne peuvent pas être vides")

                if prix_achat < 0 or prix_vente < 0 or seuil_min < 0:
                    raise ValueError("Les valeurs numériques doivent être positives")

                if ref in self._stock:
                    # Produit existant → mise à jour prix et seuil
                    existant = self._stock.get_produit(ref)
                    produit_maj = Produit(
                        ref=existant.ref, nom=nom, categorie=categorie,
                        prix_achat=prix_achat, prix_vente=prix_vente,
                        qte=existant.qte,   # ← on conserve la qte en stock !
                        seuil_min=seuil_min,
                    )
                    self._stock.mettre_a_jour_produit(produit_maj)
                    rapport["mis_a_jour"] += 1
                else:
                    # Nouveau produit — qte=0 (à réceptionner via entrée stock)
                    nouveau = Produit(
                        ref=ref, nom=nom, categorie=categorie,
                        prix_achat=prix_achat, prix_vente=prix_vente,
                        qte=0, seuil_min=seuil_min,
                    )
                    self._stock.ajouter_produit(nouveau)
                    rapport["importes"] += 1

            except (ValueError, KeyError) as e:
                rapport["erreurs"].append(f"Ligne {numero_ligne} rejetée : {e}")
                continue   # une ligne invalide ne stoppe pas l'import

    return rapport
```

**Points clés :**

```
1. encoding="utf-8-sig"
   → Lit le BOM (Byte Order Mark) si présent (exports Excel)
   → Sans le "-sig", le BOM apparaît comme "\ufeffref" dans fieldnames

2. Validation fieldnames avant itération
   → Si colonnes manquantes : ErreurCsv immédiate (fail fast)
   → Évite KeyError silencieux au milieu de milliers de lignes

3. ligne["prix_achat"].replace(",", ".")
   → CSV européens utilisent "," comme séparateur décimal
   → "2,50" → "2.50" → float(2.50) ✅

4. ref in self._stock  (utilise __contains__ de StockService V2)
   → import idempotent : relancer 2 fois le même CSV = même résultat

5. qte=existant.qte lors d'une mise à jour
   → Le fournisseur ne connaît pas notre stock physique
   → On préserve la quantité réelle, on met à jour prix/seuil uniquement

6. continue dans except
   → Une ligne invalide est logguée mais ne bloque pas les suivantes
   → Rapport d'erreurs complet à la fin
```

### 3.3 Export catalogue — `exporter_catalogue()`

```python
@journaliser("export_catalogue_csv")
def exporter_catalogue(self, chemin: str | Path | None = None) -> int:
    if chemin is None:
        horodatage = datetime.now().strftime("%Y%m%d_%H%M%S")
        chemin = CSV_DIR / f"catalogue_{horodatage}.csv"

    chemin = Path(chemin)
    chemin.parent.mkdir(parents=True, exist_ok=True)

    produits = self._stock.lister_tous()

    with open(chemin, "w", newline="", encoding="utf-8-sig") as f:
        #                   ↑                    ↑
        #            obligatoire Windows    BOM pour Excel
        writer = csv.DictWriter(f, fieldnames=COLONNES_CATALOGUE_EXPORT)
        writer.writeheader()   # ← ne pas oublier !

        for p in sorted(produits):   # tri alphabétique via __lt__ de Produit
            writer.writerow({
                "ref"       : p.ref,
                "nom"       : p.nom,
                "categorie" : p.categorie,
                "prix_achat": p.prix_achat,
                "prix_vente": p.prix_vente,
                "qte"       : p.qte,
                "seuil_min" : p.seuil_min,
            })

    return len(produits)
```

**`newline=""` — pourquoi c'est obligatoire sous Windows :**

```
Sans newline="" :
  Python ouvre en mode texte → traduit "\n" en "\r\n"
  csv.writer ajoute aussi "\r\n" (standard RFC 4180)
  Résultat : "\r\r\n" → ligne vide entre chaque ligne dans Excel

Avec newline="" :
  Python ne traduit rien → csv.writer seul gère les fins de ligne
  Résultat : "\r\n" correct → fichier propre dans Excel
```

### 3.4 Export mouvements — `exporter_mouvements()`

```python
@journaliser("export_mouvements_csv")
def exporter_mouvements(self, chemin: str | Path | None = None) -> int:
    # ... (même pattern que exporter_catalogue) ...

    mouvements = self._stock.get_mouvements()

    with open(chemin, "w", newline="", encoding="utf-8-sig") as f:
        writer = csv.DictWriter(f, fieldnames=COLONNES_MOUVEMENTS)
        writer.writeheader()

        for m in mouvements:
            try:
                nom_produit = self._stock.get_produit(m.ref).nom
            except KeyError:
                nom_produit = m.ref   # produit supprimé depuis → fallback sur ref

            writer.writerow({
                "date"   : m.date.strftime("%Y-%m-%d %H:%M:%S"),
                "type"   : m.type_mvt,
                "ref"    : m.ref,
                "produit": nom_produit,
                "qte"    : m.qte,
                "note"   : m.note,
            })

    return len(mouvements)
```

### 3.6 Prévisualisation — `previsualiser_csv()`

```python
def previsualiser_csv(self, chemin: str | Path, nb_lignes: int = 5) -> dict:
    """Lit les premières lignes d'un CSV pour prévisualiser avant import."""
    chemin = Path(chemin)
    resultat = {"colonnes": [], "lignes": [], "total_col": 0, "valide": False}

    if not chemin.exists():
        return resultat

    with open(chemin, "r", encoding="utf-8-sig") as f:
        lecteur = csv.DictReader(f)
        colonnes = list(lecteur.fieldnames or [])
        resultat["colonnes"]  = colonnes
        resultat["total_col"] = len(colonnes)

        # Vérifie si toutes les colonnes requises sont présentes
        requises = set(COLONNES_CATALOGUE_IMPORT)
        resultat["valide"] = requises.issubset(set(colonnes))

        for i, ligne in enumerate(lecteur):
            if i >= nb_lignes:
                break                  # on lit seulement nb_lignes lignes
            resultat["lignes"].append(dict(ligne))

    return resultat
```

**Pourquoi prévisualiser avant d'importer ?**
- UX : l'utilisateur voit les données avant de valider
- Validation visuelle : mauvais encodage ? colonnes manquantes ?
- Pas de modification du stock pendant la prévisualisation

### 4.3 Gestion des erreurs UI

```python
def _importer(self):
    try:
        rapport = self._csv_svc.importer_catalogue(self._chemin_import)
        # ... afficher le rapport ...

    except FileNotFoundError as e:
        # Fichier supprimé entre la prévisualisation et l'import
        messagebox.showerror("Fichier introuvable", str(e))

    except ErreurCsv as e:
        # Structure invalide détectée à l'import
        messagebox.showerror("Erreur structure CSV", str(e))
        self._ecrire_rapport(f"❌ Erreur : {e}")

    except Exception as e:
        # Erreur inattendue (disque plein, droits, etc.)
        messagebox.showerror("Erreur", f"Erreur inattendue :\n{e}")
```

**Hiérarchie des erreurs CSV :**
```
Exception
└── FileNotFoundError     ← fichier absent
└── ErreurCsv             ← colonnes manquantes (notre exception custom)
└── ValueError            ← conversions de types échouées (dans CsvService)
└── Exception             ← cas non prévus (disque plein, permissions...)
```

## 7. Bonnes pratiques CSV — résumé opérationnel

```python
import csv
from pathlib import Path

# ✅ 1. Toujours encoding="utf-8-sig" pour la compatibilité Excel
with open("fichier.csv", "r", encoding="utf-8-sig") as f:
    lecteur = csv.DictReader(f)

# ✅ 2. Toujours newline="" à l'écriture sous Windows
with open("sortie.csv", "w", newline="", encoding="utf-8-sig") as f:
    writer = csv.DictWriter(f, fieldnames=["ref", "nom"])
    writer.writeheader()   # ← ne jamais oublier !

# ✅ 3. Valider fieldnames AVANT d'itérer
colonnes_requises = {"ref", "nom", "categorie", "prix_achat"}
manquantes = colonnes_requises - set(lecteur.fieldnames or [])
if manquantes:
    raise ErreurCsv(f"Colonnes manquantes : {manquantes}")

# ✅ 4. DictReader pour l'accès par nom (résistant aux réordonnements)
for ligne in lecteur:
    ref = ligne["ref"]   # ✅ stable
    # ref = ligne[0]     # ❌ fragile si ordre change

# ✅ 5. Convertir dans un try/except, jamais directement
try:
    prix = float(ligne["prix_achat"].replace(",", "."))
    seuil = int(ligne["seuil_min"])
except ValueError as e:
    erreurs.append(f"Ligne {num} : {e}")
    continue   # ← on continue, pas de plantage global

# ✅ 6. strip() sur les champs texte
ref = ligne["ref"].strip()   # " CRAY-001 " → "CRAY-001"

# ✅ 7. Chemin par défaut avec horodatage si non fourni
if chemin is None:
    chemin = CSV_DIR / f"export_{datetime.now().strftime('%Y%m%d_%H%M%S')}.csv"

# ✅ 8. mkdir(parents=True, exist_ok=True) avant d'écrire
Path(chemin).parent.mkdir(parents=True, exist_ok=True)
```

**Ce que V8 introduit de nouveau :**
- Premier contact avec l'**acquisition de données externes** (fichiers tiers)
- Pattern **import idempotent** : même résultat si on relance (pas de doublons)
- **Exception custom** (`ErreurCsv`) pour distinguer erreurs métier vs système
- **Prévisualisation avant action** : principe UX important
- **Rapport de traitement** retourné par la méthode (pas juste un bool)

---

> ✅ **Récapitulatif V8**
>
> `CsvService` utilise `csv.DictReader` (import) et `csv.DictWriter` (export) avec `encoding="utf-8-sig"` pour Excel et `newline=""` pour Windows. La validation des `fieldnames` avant traitement garantit un **fail fast** propre. L'import est **idempotent** (relancer = même résultat), conserve la `qte` physique lors des mises à jour, et retourne un **rapport détaillé** plutôt qu'un simple booléen. L'UI offre une **prévisualisation** avant import et un **journal** des exports générés.
